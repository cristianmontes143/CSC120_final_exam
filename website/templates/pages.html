Logistic Regression

Logistic regression is a supervised learning algorithm used to predict binary or categorical output variables based on input variables.

Logistic Regression Code Documentation

The Logistic Regression

The Logical regression model is used to create a binary predictor on the possibility of heart disease it is based on heart diseases dataset the contains 5000 unique data. The model trains the data by iteratively updating the model's coefficients using a technique called maximum likelihood estimation. It uses a logistic function to map the linear combination of the input variables to a probability, which is then used to classify the data into different classes. Its main feature is the classification of the inputs of whether it has the possibility of having heart diseases or not.

Usage Instructions

To properly use the code you must enter the neccesary inputs on the UI or follow the colab link for the full code

  1. Provide the required inputs by setting the values for the following fields:
    • Age: Enter the age of the patient using the age_field input.

    • Resting Blood Pressure: Enter the resting blood pressure of the patient in mm Hg using the trtbps_field input.
    • Cholesterol: Enter the cholesterol level of the patient in mg/dl using the chol_field input.
    • Fasting Blood Sugar: Select the fasting blood sugar level of the patient (<120 mg/dl or >120 mg/dl) using the fbs_field input.
    • Resting ECG: Choose the restingographic results from the options provided using the restecg_field input.
    • Maximum Heart Rate Achieved: Enter the maximum heart rate achieved by the patient using the thalachh_field input.
    • Exercise Induced Angina: Select whether the patient experienced exercise-induced angina (No or Yes) using the exng_field input.
    • ST Depression Induced by Exercise: Enter the ST depression induced by exercise value using the oldpeak_field input.
    • Slope of the Peak Exercise ST Segment: Choose the slope of the peak exercise ST segment from the options provided using the slp_field input.
    • Number of Major Vessels Colored by Flourosopy: Select the number of major vessels (0-3) colored by flourosopy using the caa_field input.
    • Thalassemia: Choose the thalassemia type from the options provided using the thall_field input.
  2. After providing the inputs, run the code and wait for the prediction to be generated. The possible outputs are as follows:
    • If the output is 0, the model predicts that there is a risk of heart disease. This will be displayed as "The model predicts that there is a risk of heart disease."
    • If the output is not 0, the model predicts that there is no risk of heart disease. This will be displayed as "The model predicts that there is no risk of heart disease."

Heart Disease Diagnosis Datasets: Detailed Information

This collection includes four datasets related to heart disease diagnosis, sourced from esteemed medical institutions: Cleveland Clinic Foundation, Hungarian Instiology, V.A. Reinforcement Learning

Reinforcement learning is a type of machine learning where an agent learns to make decisions in an environment, based on trial-and-error learning and feedback in the form of rewards or penalties.

Reinforcement Learning Code Documentation

The Reinforcement Learning

The code aims to simulate a traffic gridworld environment where vehicles move and interact within a predefined grid. It utilizes reinforcement learning techniques, specifically Q-learning, to train an agent to navigate the traffic environment and maximize rewards.

Usage Instructions

The Reinforcement Learning Model doesn't have a working User Interface instead we video of the agent on the gridworld.

Code functionalities

  1. The code defines a TrafficGridworldEnv class that represents the traffic gridworld environment. It initializes the environment with a specified width, height, number of cars, and number of steps.
  2. The environment consists of different road structures such as intersections, roundabouts, 2+1 roads, and highways. Each structure is represented by a set of coordinates on the grid
  3. The step function in the environment class allows the agent to take actions and updates the state accordingly. It also handles collision detection and provides rewards based on the current state and action
  4. The reset function resets the environment to its initial state.
  5. The render function displays a visual representation of the gridworld environment.

Overall, this code provides a framework for simulating and training an agent to navigate a traffic gridworld environment using reinforcement learning techniques.

Code's Important Functions

Initializes the TrafficGridworldEnv class with the specified parameters. It sets the width and height of the grid, the number of cars in the environment, and the number of steps for each episode.

TrafficGridworldEnv.step(self, action): Takes an action as input and updates the state of the environment accordingly. It moves the current car or changes the current state based on the action. It checks for collisions and determines the reward and termination status of the current state.

TrafficGridworldEnv.reset(self): Resets the environment to its initial state. It reinitializes the grid and randomly assigns positions to the cars.

Vehicle.__init__(self, length, speed, direction): Initializes the Vehicle class with the specified parameters. It sets the length, speed, and direction of the vehicle.

Vehicle.move(self, grid): Moves the vehicle in the specified direction if it is possible, considering collisions with other vehicles and grid boundaries.

Vehicle.can_move(self, grid): Checks if the vehicle can move in the specified direction without colliding with other vehicles or going outside the grid.

generate_vehicles(grid): Generates a random number of vehicles in the gridworld environment. It creates vehicles with random types, positions, speeds, and directions.

run_traffic_simulation(env, q_table): Runs the traffic simulation using the trained Q-table. It iterates over episodes and performs actions based on the Q-values, updating the rewards and success rates.

get_state(env): Retrieves the state of the environment by combining information about the current state and car. It returns a unique state value.

render_animation(env, q_table): Renders an animation of the traffic simulation. It displays the movement of vehicles in the gridworld environment using OpenCV.

Classes

  • TrafficGridworldEnv: Represents the traffic gridworld environment. It has properties such as width, height, num_cars, and num_steps. It includes methods for stepping through the environment, resetting it, and rendering a visual representation.
  • Vehicle: Represents a vehicle in the traffic environment. It has properties such as length, speed, and direction. It includes methods for moving the vehicle, checking if it can move, and detecting collisions.

Variables

  • width: Represents the width of the gridworld environment.
  • height: Represents the height of the gridworld environment.
  • num_cars: Represents the number of cars in the environment.
  • num_steps: Represents the number of steps for each episode.
  • q_table: Represents the Q-table used for Q-learning.
  • rewards: Dictionary mapping specific states to corresponding rewards.
